## Review
- Correct: The prompt does not auto-enable reversal from detection alone. Detection is only used to compute inline/onboarding suggestions (`Dockmint/ContentView.swift:115-118`, `Dockmint/OnboardingView.swift:105-123`), and the preference is only set to true from the explicit Turn On action path (`Dockmint/Preferences.swift:1236-1238`).
- Correct: Dismiss/seen persistence is backed by UserDefaults arrays and suppresses future suggestions as intended (`Dockmint/Preferences.swift:1054-1069`, `Dockmint/Preferences.swift:1217-1234`). Focused persistence tests cover dismissal, seen, and Turn On behavior (`DockmintTests/PreferencesDefaultsXCTest.swift:100-130`).
- Correct: Reverse scroll direction remains user-controlled in event handling. `shouldInvertDiscreteScrollDirection` now returns true only for non-continuous events when the user preference is true (`Dockmint/DockDecisionEngine.swift:297-300`), and `DockClickEventTap` reads only `invertDiscreteScrollDirection` rather than remapper detection (`Dockmint/DockClickEventTap.swift:303-314`). Continuous/trackpad events are still excluded by the engine tests (`DockmintTests/DockDecisionEngineXCTest.swift:376-397`, `DockmintTests/DockDecisionEngineXCTest.swift:528-560`).
- Correct: UI copy is explicit and avoids overclaiming that the detected tool definitely reverses the current mouse; it says “If you use it or a similar app to reverse...” in Settings and onboarding (`Dockmint/ContentView.swift:424-427`, `Dockmint/OnboardingView.swift:105-110`).
- Blocker: The diff changes fresh-install scroll action defaults from `.none` to `.appExpose` and updates the install-flow script expectations accordingly (`Dockmint/Preferences.swift:801-812`, `scripts/automated_default_install_flow_checks.sh:123-124`, `scripts/automated_default_install_flow_checks.sh:203-204`). This is outside the approved prompt plan and creates a broad behavior change: new users would get Dock scroll actions enabled by default. Unless this was separately approved, revert these default/script changes so the prompt work stays minimal and does not introduce accidental scroll behavior.
- Blocker: `resetAppActionsToDefaults()` now silently clears the general `reverseMouseScrollActions` checkbox (`Dockmint/Preferences.swift:1817-1842`). That button is labeled “Reset App Actions” in Settings (`Dockmint/ContentView.swift:323-325`, `Dockmint/ContentView.swift:522-524`), not “reset general scroll direction,” so a user who explicitly enabled reverse mouse scrolling can lose the setting while resetting action mappings. This violates persistence/manual-control expectations for the new checkbox and is not covered by tests.
